# contributing

mycel is in design-first mode. no production source directories should be added until the initial ADRs are settled. **confidence: solid. load-bearing.**

## basics

- use conventional commits, for example `docs(adr): choose substrate format`. **confidence: directional.**
- keep claims confidence-tagged as **solid**, **directional**, or **vibes**. **confidence: directional. load-bearing.**
- never ship vibes-tier claims as facts. **confidence: directional. load-bearing.**
- prefer local-first designs. cloud services need a written reason. **confidence: directional. load-bearing.**
- update ADRs when a design decision changes. **confidence: directional.**
- keep public prose direct, specific, and low-gloss. **confidence: directional.**

## design changes

for now, design changes should include:

1. the problem. **confidence: directional.**
2. the proposed decision. **confidence: directional.**
3. tradeoffs. **confidence: directional.**
4. confidence tags. **confidence: directional. load-bearing.**
5. affected roadmap or ADR files. **confidence: directional.**

## code changes

code changes are intentionally out of scope until the project exits initialization. **confidence: solid. load-bearing.**
